# FIFA-SQL-Analysis
<I>Find transfer value faster with my SQL based analysis system on 18,000+ players!</I>


<B>Overview:</B>

Built a SQL-based analysis project to identify useful FIFA player market insights across a dataset of 18,000+ players.

 - The goal was to move beyond manual filtering and use SQL to make player scouting, market comparison, and transfer shortlisting more structured.

<B>What it does:</B>

 - The SQL script analyzes key transfer factors:

   - Position

   - Age group

   - Market value

   - Wage budget

   - Overall rating

   - Potential rating

   - Club

   - Nationality


   The queries group, filter, and rank players to return useful insights such as top players by position, average value by age group, wage efficiency by club, and shortlists of young affordable players with strong potential.

<B>Why it matters:</B>
 - Instead of manually comparing thousands of rows, this project turns player scouting into a structured SQL analysis process--similar to how an analyst would explore data for a BI dashboard or transfer report.

<B>Skills demonstrated:</B>

 - Data quality checks in SQL

 - Creating a reusable analysis view

 - Using CASE statements for age-band grouping

 - Aggregating data by position, club, and nationality

 - Using CTEs for comparison analysis

 - Using RANK() to compare players within each position

 - Building transfer shortlists with real-world criteria
